Single-page scroll vs. multi-page routing?

Both have the same sticky nav — the difference is what happens when you click a nav link.

A

Single-page scroll (recommended)

All sections live on one page, stacked vertically. Clicking a nav link smoothly scrolls to that section. URL can optionally update to #projects, #skills, etc.

Pros

  • Matches the sawad.framer.website feel exactly
  • Smooth, editorial scroll experience
  • No page loads or flicker
  • Simpler code — no router needed

Cons

  • All content loads at once (negligible for a portfolio)
B

Multi-page routing (React Router)

Each section is its own route: /projects, /skills, etc. Clicking nav links navigates to a new "page" (still client-side, no full reload).

Pros

  • Clean URLs per section
  • Can deep-link directly to any page

Cons

  • Loses the smooth scroll feel
  • Requires React Router + GitHub Pages hash routing workaround
  • Feels more like an app, less like an editorial portfolio
