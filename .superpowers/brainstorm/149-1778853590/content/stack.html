Which tech stack for your portfolio?

All three deploy to GitHub Pages as static files. Pick based on your comfort and maintenance preference.

A

Single HTML File

One index.html with embedded CSS & JS. Open in browser directly, push to GitHub, done. No build step, no Node.js required.

Pros

  • Zero setup
  • Edit & preview instantly
  • Deploy by pushing one file

Cons

  • File gets long (600–900 lines)
  • No component reuse
B

React + Vite (recommended)

Component-based project with npm run build producing a static dist/ folder. Deploy that folder to GitHub Pages via gh-pages action.

Pros

  • Each section is its own component
  • Easy to update content in isolation
  • Scales well as site grows
  • Hot-reload during dev

Cons

  • Requires Node.js + npm
  • Build step before deploy
C

Next.js (static export)

Full Next.js project with output: 'export'. More powerful, but heavier than needed for a portfolio.

Pros

  • File-based routing feels natural
  • Good ecosystem

Cons

  • Overkill for a portfolio
  • More config for GitHub Pages
  • Slower build
